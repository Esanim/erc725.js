encodeData

  • encodeData(data: {}): {}
    • Defined in index.ts:247

    To be able to store your data on the blockchain, you need to encode it according to your ERC725JSONSchema.

    Parameters

    • data: {}

      An object with one or many properties, containing the data that needs to be encoded.

      • [key: string]: any

    Returns {}

    An object with the same keys as the object that was passed in as a parameter containing the encoded data, ready to be stored on the blockchain.

    :::tip When encoding JSON it is possible to pass in the JSON object and the URL where it is available publicly. The JSON will be hashed with keccak256. :::

fetchData

  • fetchData(keyOrKeys?: string | string[]): Promise<{}>
    • Defined in index.ts:174

    Since getData exclusively returns data that is stored on the blockchain, fetchData comes in handy. Additionally to the data from the blockchain, fetchData also returns data from IPFS or HTTP(s) endpoints stored as JSONURL or ASSETURL.

    :::info To ensure data authenticity fetchData compares the hash of the fetched JSON with the hash stored on the blockchain. :::

    Parameters

    • Optional keyOrKeys: string | string[]

      The name (or the encoded name as the schema ‘key’) of the schema element in the class instance’s schema.

    Returns Promise<{}>

    Returns the fetched and decoded value depending ‘valueContent’ for the schema element, otherwise works like getData

getData

  • getData(keyOrKeys?: string | string[]): Promise<{}>
    • Defined in index.ts:144

    Gets decoded data for one, many or all keys of the specified ERC725 smart-contract. When omitting the keyOrKeys parameter, it will get all the keys (as per ERC725JSONSchema definition).

    :::caution Data returned by this function does not contain external data of JSONURL or ASSETURL schema elements.

    If you would like to receive everything in one go, you can use fetchData for that. :::

    Parameters

    • Optional keyOrKeys: string | string[]

    Returns Promise<{}>

    An object with schema element key names as properties, with corresponding decoded data as values.
